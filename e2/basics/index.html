Expression 2 is a tool that lets you programmatically interact with the other tools provided in the wiremod addon, and the game itself.

Let's start with history

All the historical information is gathered from the archived wiremod wiki and the archived wiremod forums. Take a dive if you're interested in what else was going on in the past.

Once there was Expression 1...

Expression 1 was released around 2007 - 2008. Coding in Expression 1 looked like this:

Looks painful, if you ask me. First of all, no colors. On top of that you have to physically click the next box to add a new line. And let's not speak of debugging (Don't worry, I'll explain strange words like these afterwards). However, times have changed. Expression 1 wasn't used a lot after its successor was released, and got removed from Wiremod entirely in 2010

...then came Expression 2

The differences between Expression 1 and Expression 2 are just too much to explain in detail. What I can say is that Expression 2 got its own editor, nicer syntax, added language features, APIs and more.

Expression 2 has had a lot of changes since its release in 2009, mostly under the hood.

Under the hood; a metaphorical area that contains the underlying implementation of something - e.g. a piece of hardware, a piece of software, an idea, etc. - Online Slang Dictionary

...or in plain English, the parts of Expression 2 you can't see or interact with.

But why?

Expression 2 is a tool that lets you programmatically interact with the other tools provided in the wiremod addon, and the game itself. You can get player data, write something to a screen, make confetti burst out while kids are screaming and party horns are blasting. To be clear, these aren't features that are unique to Expression 2. Using gates, buttons, constant values and screens, you can already start working on big things like showing player stats on a screen. However, Expression 2 makes it way easier to build advanced contraptions that had looked like a giant pile of gates and wiring and whatnot if it were built without Expression 2.

Expression 2 code looks like this:

@name Hello World
@inputs A B
@outputs Add Sub Mul Div
@outputs GreaterThan Highest Lowest
@outputs Vector:vector
@persist D
@trigger all

Add = A + B
Sub = A - B
Mul = A * B
Div = A / B

GreaterThan = A > B

if(A > B) {
	Highest = A, Lowest = B
} else {
	Highest = B, Lowest = A
}

Vector = vec(A, B, 0)
Vector = Vector + vec(0, 0, A + B)
Vector = Vector:normalized()
